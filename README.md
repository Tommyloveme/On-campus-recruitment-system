# 校招候选人管理系统

跨平台（Windows / SUSE Linux）的校招候选人跟踪管理系统，支持 Excel 导入、字段配置、权限分组管理与完整的修改日志。

## 功能特性

- **候选人全流程跟踪**：候选人、电话、接口人经理、接口人、入职二层/三层、Offer 状态、签约状态、拟录取工作地、毕业时间、预计入职时间、预约体检时间、是否体检、是否预约入职、预约入职时间、是否入职、入职风险等。
- **Excel 批量导入**：上传 `.xlsx` 即可导入/更新候选人（按电话或姓名自动匹配去重），可下载标准导入模板。
- **筛选与排序**：表头每列都有独立筛选条件（文本列模糊匹配、状态列下拉精确匹配），可单列或多列任意组合（AND）；日期列（毕业时间、预计入职时间、预约体检/入职时间）点击表头即可升序/降序排序，空值始终排在最后。
- **简历管理**：每位候选人可上传一份简历（支持 `.pdf` / `.docx`），随时下载、更换、删除；所有简历操作均记录日志。
- **批量导出简历**：列表支持多选（含"全选当前筛选结果"），一键将选中候选人的简历打包为 zip 下载，文件以"姓名_原文件名"命名。
- **JSON 配置驱动**：`config/fields.json` 统一配置每个字段的显示名、Excel 表头、类型、下拉选项，以及"是否导入 / 是否网页显示 / 是否可编辑"；管理员也可在「系统管理」页面直接勾选网页显示列。
- **权限管理与分组**：
  - 角色四级：`系统管理员`（全部权限）、`组管理员`（本组候选人增/删/改/查 + 自行添加本组成员）、`组成员`（本组增/改/查，**无删除权限**）、`只读`（仅查看本组）；
  - 每个用户归属一个权限分组，普通用户只能看到本组候选人；
  - 组管理员有独立的「成员管理」页，可向本组添加组成员/只读账号（不能创建管理员）；
  - 系统管理员可在「全局总览」一个页面查看所有分组的候选人及其**最新进展**（最近一次操作记录）与统计（已签约/已入职/高风险）。
- **数据透视图表**（系统管理员）：类 Excel 数据透视图，自由选择分组范围、维度列（横轴）、系列列（图例/堆叠），支持柱状图、堆叠柱状图、条形图、环形图、饼图，旁边同步给出数据透视表；Chart.js 已离线内置，无需联网。
- **修改留痕**：任何新增、修改、删除、导入都自动记录日志，用简洁语言呈现，如：
  > 招聘专员-小王 修改了「张伟」：签约状态：未签约 → 已签约；入职风险：中 → 低

## 快速开始

依赖：Python 3.9+（Windows 与 SUSE Linux 均可）。

**Windows：**

```bat
start.bat
```

**SUSE Linux / 其他 Linux：**

```bash
chmod +x start.sh
./start.sh
```

脚本会自动创建虚拟环境、安装依赖并启动服务，随后浏览器访问 `http://127.0.0.1:8000`。

- 默认管理员账号：`admin / admin123`（请登录后及时新建账号并修改）
- 如需写入演示数据（示例分组/用户/候选人）：`start.bat --demo` 或 `./start.sh --demo`，演示账号：`lead01`（组管理员）、`hr01`、`hr02`（组成员），密码均为 `123456`
- 端口可通过环境变量 `PORT` 修改。

## Excel 导入说明

1. 在「候选人」页点击 **下载导入模板**，获得与当前字段配置一致的表头；
2. 填好数据后点击 **Excel 导入**，选择目标权限分组并上传；
3. 已存在的候选人（电话或姓名相同）将被更新，其余新增，结果会写入操作日志。

## 字段配置（config/fields.json）

每个字段支持以下属性：

| 属性 | 说明 |
| --- | --- |
| `key` | 内部字段名 |
| `label` | 网页显示名称 |
| `excel_column` | Excel 导入时匹配的表头 |
| `type` | `text` / `date` / `select` |
| `options` | 下拉选项（type 为 select 时） |
| `visible` | 是否在网页表格中显示 |
| `editable` | 是否允许手动编辑 |
| `importable` | 是否参与 Excel 导入 |
| `required` | 新增时是否必填 |

修改该文件后无需重启服务，刷新页面即可生效；新增字段只需在此文件中追加一项。

## 项目结构

```
├── app.py               # Flask 后端（认证/权限/候选人/导入/日志/总览）
├── config/fields.json   # 字段配置（导入列 + 网页显示列）
├── static/              # 前端页面（原生 HTML/CSS/JS，无构建依赖）
├── data/                # SQLite 数据库 + 简历文件（首次运行自动创建）
├── start.bat            # Windows 启动脚本
├── start.sh             # Linux (SUSE) 启动脚本
└── requirements.txt
```
