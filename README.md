# 校招入职跟踪管理系统

跨平台（Windows / SUSE Linux）的校招入职跟踪管理系统，面向**已接收 Offer** 的校招候选人，跟踪签约、体检、入职预约到正式入职的全流程，支持 Excel 导入、字段配置、权限分组管理与完整的修改日志。

## 功能特性

- **入职全流程跟踪**：三层部门、候选人、电话、学历、毕业院校、专业、接口人、Offer 状态、签约状态、拟录取工作地、毕业时间、预计入职时间、预约体检时间、是否体检、是否预约入职、预约入职时间、是否入职、入职风险、当前进展等（学历/毕业院校/专业/电话默认隐藏，可在字段配置中开启）。
- **当前进展跟踪**：进展列带「更新」快捷按钮，新内容自动写在**第一行**并带今天的日期前缀（如 `0612：`），列表始终显示第一行（最新进展），完整历史悬浮可见；新增候选人时也自动预填。
- **批量删除**：勾选候选人后一键批量删除（仅系统管理员/组管理员可见，组管理员限本组）。
- **高并发支持**：waitress 多线程 WSGI 服务器 + SQLite WAL 模式（读写不互斥），支撑 500+ 人同时在线使用。
- **分页浏览**：候选人列表默认每页 15 人，可切换 15/30/50/100/全部；过长内容自动省略号截断、悬浮显示完整信息；**所有列宽可拖拽调整**。
- **运行配置化**：端口、线程数、上传上限、分页大小、默认密码等易变内容集中在 `config/app_config.json`。
- **选中导出**：勾选候选人后可一键导出 Excel 数据或打包下载简历 zip。
- **操作手册**：网页右上角「操作手册」可查看 PDF 版详细使用说明（`tools/gen_manual.py` 可重新生成）；维护与测试参见 `docs/维测指导手册.md`。
- **Excel 批量导入**：上传 `.xlsx` 即可导入/更新候选人（按电话或姓名自动匹配去重），可下载标准导入模板。
- **筛选与排序**：表头每列都有独立筛选条件（文本列模糊匹配、状态列下拉精确匹配），可单列或多列任意组合（AND）；日期列（毕业时间、预计入职时间、预约体检/入职时间）点击表头即可升序/降序排序，空值始终排在最后。
- **简历管理**：每位候选人可上传一份简历（支持 `.pdf` / `.docx`），支持**在线预览**（PDF 浏览器内嵌阅读、DOCX 自动转网页排版）、下载、更换、删除；所有简历操作均记录日志。
- **批量导出简历**：列表支持多选（含"全选当前筛选结果"），一键将选中候选人的简历打包为 zip 下载，文件以"姓名_原文件名"命名。
- **JSON 配置驱动**：`config/fields.json` 统一配置每个字段的显示名、Excel 表头、类型、下拉选项，以及"是否导入 / 是否网页显示 / 是否可编辑"；管理员也可在「系统管理」页面直接勾选网页显示列。
- **权限管理与分组**：
  - 角色五级：`系统管理员`（全部权限）、`全局查看员`（界面与管理员一致——全分组候选人/全局总览/数据图表/日志，但**只读**且无系统管理入口）、`组管理员`（本组候选人增/删/改/查 + 自行添加本组成员）、`组成员`（本组增/改/查，**无删除权限**）、`只读`（仅查看本组）；
  - 每个用户归属一个权限分组，普通用户只能看到本组候选人；
  - 组管理员有独立的「成员管理」页，可向本组添加组成员/只读账号（不能创建管理员）；
  - 系统管理员可在「全局总览」一个页面查看所有分组的候选人及其**最新进展**（最近一次操作记录）与统计（已签约/已入职/高风险）。
- **数据透视图表**（系统管理员 + 组管理员）：类 Excel 数据透视图，自由选择分组范围、维度列（横轴）、系列列（图例/堆叠），支持柱状图、堆叠柱状图、条形图、环形图、饼图，旁边同步给出数据透视表；日期列可按**年月日 / 年月 / 月份**三种粒度聚合呈现；组管理员自动限定本组数据；Chart.js 已离线内置，无需联网。
- **按组字段显示配置**：全局默认配置存于 `config/fields.json`，每个分组可有独立配置文件 `config/fields_group_<id>.json` 覆盖显示开关；系统管理员可配置全局或任意分组，组管理员可在「成员管理」页配置本组的列显示。
- **默认密码**：管理员/组管理员添加成员时默认密码为 `123456`（可修改）。
- **修改留痕**：任何新增、修改、删除、导入都自动记录日志，用简洁语言呈现，如：
  > 招聘专员-小王 修改了「张伟」：签约状态：未签约 → 已签约；入职风险：中 → 低

## 快速开始

依赖：Python 3.9+（Windows 与 SUSE Linux 均可）。

**Windows：**

```bat
start.bat
```

**SUSE Linux / 其他 Linux：**

```bash
chmod +x start.sh
./start.sh
```

脚本会自动创建虚拟环境、安装依赖并启动服务，随后浏览器访问 `http://127.0.0.1:8000`。

- 默认管理员账号：`admin / admin123`（请登录后及时新建账号并修改）
- 如需写入演示数据（示例分组/用户/候选人）：`start.bat --demo` 或 `./start.sh --demo`，演示账号：`lead01`（组管理员）、`hr01`、`hr02`（组成员），密码均为 `123456`
- 端口可通过环境变量 `PORT` 修改。

## Excel 导入说明

1. 在「候选人」页点击 **下载导入模板**，获得与当前字段配置一致的表头（首列「三层部门」，其次「候选人」等；**二层部门**对应权限分组，在导入时选择，不在 Excel 中）；
2. 填好数据后点击 **Excel 导入**，选择目标权限分组并上传；
3. 已存在的候选人（电话或姓名相同）将被更新，其余新增，结果会写入操作日志。

## 字段配置（config/fields.json）

每个字段支持以下属性：

| 属性 | 说明 |
| --- | --- |
| `key` | 内部字段名 |
| `label` | 网页显示名称 |
| `excel_column` | Excel 导入/导出时匹配的表头 |
| `excel_aliases` | 旧表头别名（兼容历史 Excel，如「入职三层」→ 三层部门） |
| `type` | `text` / `date` / `select` |
| `options` | 下拉选项（type 为 select 时） |
| `visible` | 是否在网页表格中显示 |
| `editable` | 是否允许手动编辑 |
| `importable` | 是否参与 Excel 导入 |
| `required` | 新增时是否必填 |

修改该文件后无需重启服务，刷新页面即可生效。**该文件是字段的唯一数据源**：新增字段只需在此追加一项，Excel 导入模板、导入匹配、网页表格列、编辑表单、数据图表维度全部自动联动，无需改代码或数据库。

## 维测脚本（scripts/）

| 命令 | Windows | Linux | 说明 |
| --- | --- | --- | --- |
| 状态 | `scripts\status.bat` | `./scripts/status.sh` | 端口/PID/残留进程检测 |
| 停止 | `scripts\stop.bat` | `./scripts/stop.sh` | 按端口定位并停止服务 |
| 强制停止 | `scripts\force_stop.bat` | `./scripts/force_stop.sh` | 杀掉端口进程+所有 app.py 残留进程 |
| 重启 | `scripts\restart.bat` | `./scripts/restart.sh` | 停止后后台启动（日志写 `data/server.log`） |

## 项目结构

```
├── app.py               # Flask 后端（认证/权限/候选人/导入导出/简历/图表/日志/总览）
├── config/app_config.json      # 运行配置（端口/线程/分页/默认密码等）
├── config/fields.json   # 全局字段配置（导入列 + 网页显示列 + 徽章颜色）
├── config/fields_group_*.json  # 各分组独立的字段显示配置（自动生成）
├── docs/维测指导手册.md  # 维护与测试指导
├── scripts/             # 维测脚本（status/stop/force_stop/restart，bat+sh 双平台）
├── tools/gen_manual.py  # 操作手册 PDF 生成脚本（输出 static/manual.pdf）
├── static/              # 前端页面（原生 HTML/CSS/JS，无构建依赖）
├── data/                # SQLite 数据库 + 简历文件（首次运行自动创建）
├── start.bat            # Windows 启动脚本
├── start.sh             # Linux (SUSE) 启动脚本
└── requirements.txt
```
